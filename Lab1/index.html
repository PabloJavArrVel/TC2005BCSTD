Preguntas 1) ¿Cuál es la diferencia entre Internet y la World Wide Web? El internet es la red de conexión entre computadoras, en contraste la WWW es la coleción de páginas web. (BBC News Mundo, 2019) 2) ¿Cuáles son las partes de un URL? El protocolo de internet, el dominio principal, y la ruta (New Code, 2021) 3) ¿Cuál es el propósito de los métodos HTTP: GET, HEAD, POST, PUT, PATCH, DELETE? El método put se utiliza para actualizar o insertar un recurso, post tiene la misma función, la diferencia es que put necesita el ID específico mientras que post no, delete suprime recursos y, al igual que put necesita la id específica (Métodos PUT, POST y DELETE, 2018). Patch actualiza parcialmente un recurso, get tiene una función similar que post, sin embargo, es menos segura, head también tiene la misma función que get pero sirve para ver como reaccionará get antes de que reaccione (HTTP Methods GET vs POST, s. f.) 4) ¿Qué método HTTP se debe utilizar al enviar un formulario HTML, por ejemplo cuando ingresas tu usuario y contraseña en algún sitio? ¿Por qué? Probablemente post, en mi investigación constantemente es mencionado como un método seguro. 5) ¿Qué método HTTP se utiliza cuando a través de un navegador web se accede a una página a través de un URL? El método Get 6) Un servidor web devuelve una respuesta HTTP con código 200. ¿Qué significa esto? ¿Ocurrió algún error? Es uno de los estatus que un servidor puede arrojar, significa que la respuesta es exitosa (Hypertext Transfer Protocol (HTTP) Status Code Registry, s. f.). 7) ¿Es responsabilidad del desarrollador corregir un sitio web si un usuario reporta que intentó acceder al sitio y se encontró con un error 404? ¿Por qué? No necesariamente, una respuesta 404 es arrojada cuando un recurso no se encuentra, esto quiere decir que puede ser que el recurso no exista en un primer lugar, sin embargo, si el recurso sí existe pero no se encuentra entonces sí se debe corregir (Hypertext Transfer Protocol (HTTP) Status Code Registry, s. f.). 8) ¿Es responsabilidad del desarrollador corregir un sitio web si un usuario reporta que intentó acceder al sitio y se encontró con un error 500? ¿Por qué? Sí, la respuesta significa que hubo un error interno en el servidor que el usuario intentó accesar, por tanto debe ser corregido 9) ¿Qué significa que un atributo HTML5 esté depreciado o desaprobado (deprecated)? Menciona algunos elementos de HTML 4 que en HTML5 estén desaprobados. Significa que no está soportado por muchos navegadores, es decir, cuando usuarios intenten accesar en algunos navegadores los recursos pueden no cargarse como se esperaría. Algunos ejemplos de atributos desaprobados son applet, dir y isindex (Tags y atributos desaprobados en HTML - TODO HTML, s. f.) 10) ¿Cuáles son las diferencias principales entre HTML 4 y HTML5? HTML5 tiene nuevas etiquetas, nuevo sytnax, es más fácil de utilizar y no utiliza muchas etiquetas desaprobadas (Difference between HTML and HTML5 - javatpoint, s. f.) 11) ¿Qué componentes de estructura y estilo tiene una tabla? En su estructura se encuentra table que englobará todo, tr que definirá las filas, td que definirá las celdas, th se utiliza para definir las celdas de encabezado. En cuanto estilo se tiene los clásicos widht, height, border además de estilos específicos de tabal como el cellspacing o el cellpading (W3Schools, s.f.). 12) ¿Cuáles son los principales controles de una forma HTML5? Input, textarea, checkbox, radio, button y datalist. (W3Schools, s.f.) 13) ¿Qué tan compatible es mi navegador?
